# Phases 13-16 Design: AIE Trace, Kernel Optimization, Chess Compiler, Power Characterization

**Date:** 2026-03-30
**Model:** Meta-Llama-3.1-8B-Instruct Q8_0
**Hardware:** Ryzen AI MAX 385 · XDNA2 NPU
**Baseline:** 1.40 t/s decode (128x128x16 micro-tiles, Phase 12), 0.6% AIE utilization

---

## Problem

All host-side optimizations are exhausted (Phases 10-12). Phase 12 proved micro-tile geometry matters more than dispatch reduction (+23% from tile tuning vs 0% from 7x dispatch reduction). The bottleneck is inside the AIE cores. At 0.6% of 480 GOPS theoretical peak, massive headroom exists but the root cause is unknown — compute-bound, DMA-stalled, or idle cores?

---

## Architecture Decision

Trace profiling first (Phase 13), then targeted kernel optimization (Phase 14). The alternative — jumping directly to kernel modification based on guesses — was rejected because 0.6% utilization leaves too many possible root causes. Phase 15 (Chess compiler) is gated on external license. Phase 16 is reframed as characterization (5.7 J/tok NPU vs 1.3 J/tok Vulkan makes efficiency claims untenable).

## Phase 13: AIE Trace Profiling

### 13A — Standalone Trace (Wave 1)
- Build trace-enabled xclbin via `whole_array_trace.py` (adapted from event_trace example)
- Trace events: INSTR_VECTOR, MEMORY_STALL, STREAM_STALL, LOCK_STALL, PORT_RUNNING, PORT_IDLE
- Run standalone via `whole_array.exe`, parse with `parse.py` → Chrome DevTools JSON
- Trace buffer: 8KB default (matches reference BD config at `aie_trace.py` line 168)
- Deliverables: `tools/trace/build-trace-xclbin.sh`, `tools/trace/run-trace.sh`, `tools/trace/parse-trace.sh`
- Quality gate: zero changes to non-trace dispatch path

### 13B — Backend Integration (Wave 2, conditional)
- **Gate:** Trace shows DMA stall >10%, lock stall >5%, or idle gaps >20%
- Add `xdna_env_bool()` helper near line 96 of ggml-xdna.cpp
- Enlarge bo_trace from 4 bytes to `XDNA_TRACE_BUF_SIZE` (8192) when `GGML_XDNA_TRACE=1`
- Configurable via `GGML_XDNA_TRACE_SIZE` (min 4096, max 1MB, default 8192)
- Write raw trace to `GGML_XDNA_TRACE_DIR/<timestamp>-slot<N>.bin`
- Cap file accumulation at `GGML_XDNA_TRACE_MAX_FILES` (default 100)
- Quality gate: trace disabled = identical dispatch path, +-2% t/s vs baseline

### 13 Side-Tasks (Wave 1.5, parallel)
- Submit AMD Ryzen AI Early Access application (Chess/Vitis license)
- Run one decode-mode power baseline with current bench-power.sh

## Phase 14: Kernel Optimization

### 14A — Parameter Sweep (Wave 3)
- Build `tools/sweep/sweep.sh`: fifo_depth {2,3,4} x c_row_maj {0,1} x OPT_PERF_ENABLED {0,1}
- Each combination: rebuild xclbin, run 100-token decode via llama-bench, append CSV
- Output: `tools/sweep/sweep-results.csv` (schema defined in brief)
- If trace shows purely instruction-limited: skip fifo_depth, keep OPT_PERF_ENABLED + c_row_maj
- Time: ~15 min per combination, full matrix runs overnight

### 14B — Kernel Fork (Wave 4)
- Fork `mm.cc` → `mm_decode.cc` in mlir-aie/aie_kernels/aie2p/
- Specialize for decode shape: m=128, k=128, n=16 (hardcoded)
- Remove unused dtype combinations — keep i8_i32_ONLY
- K-loop unroll 2x first (line 154), then try 4x only if 2x succeeds without register spill
- 4x2 blocking pattern (outer line 91: z+=4, inner line 101: j+=2)
- Apply best fifo_depth + c_row_maj from 14A
- Quality gate: bit-exact output for reference test matrix before any perf measurement

### 14C — Full Custom Kernel (conditional, 2-week hard time-box)
- **Gate:** 14A + 14B combined gain < 5% over Phase 12 baseline
- Explore: double-buffered A/B loads, asymmetric blocking (8x1), custom DMA scheduling
- Hard stop: 2 calendar weeks. Document findings regardless.

## Conflict Resolutions

| Conflict | Resolution | Rationale |
|----------|-----------|-----------|
| Trace buffer size | 8KB default, configurable to 1MB | Matches reference BD config; configurable for multi-tile |
| Phase 16 framing | "Characterization" | 5.7 vs 1.3 J/tok; advantage claim untenable |
| Sweep script timing | Phase 14A deliverable | The sweep IS the phase |
| bench-power.sh fixes | Phase 16A prerequisite | Phase 13 uses rough baseline |
| Phase 14A scope if no stalls | Skip fifo_depth only | OPT_PERF + c_row_maj still affect instruction scheduling |
| K-loop unroll | 2x first, 4x conditional | Register spill risk on AIE2P |
